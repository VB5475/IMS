3. Business Rules & Validations

#RuleDetail
BR-01Division is a prerequisiteUser must select Division before clicking Select Item or Select Supplier. Alert shown if not selected.
BR-02Division cascades to Inquiry TypeWhen Division changes: Inquiry Type dropdown clears immediately, then reloads options for the new Division via API.
BR-03Based On controls item picker modeDirect (0) → uses RB_PurInqSelOnlyItem; Indent wise (2) → uses RB_PurInqSelIndtItem and collapses items with indent children.
BR-04Indent wise inserts are aggregatedSelected indent rows go through Fn_tbl_FetchIndentSummaryItem4Inquiry. The response returns parent item rows; original indent rows become collapsible children.
BR-05Date format for APIAll dates sent to API must be in dd-Mon-yyyy format (e.g. 02-Jun-2026). Use formatTranDate().
BR-06Edit mode gatePage is read-only on load. Grids, filter fields, and action buttons are disabled until user clicks Add New in the action bar.
BR-07Cancel resets all stateConfirms with user, then wipes localStorage meta, resets all state, clears both grids, and forces filter panel remount.
BR-08Sr.No auto-resequence on deleteAfter deleting supplier rows, remaining rows are renumbered sequentially.
BR-09Temp IDs never sent to backendAll locally created rows carry a negative integer id. On save, id is stripped before building the JSON payload.

4.2 Load Sequence

1
Phase 1 — On Page Mount (parallel)
Fetch master RB meta (RB_PurInquiryMst) → get RBID & SaveProcName
Then in parallel: Header columns (GetDetailColData) + Division options + Department options
Simultaneously: Detail RB meta (RB_PurInquiryDet) → RBID → Detail column definitions
2
Phase 2 — On Division Select (cascade)
Fetch Inquiry Types for selected Division via fn_tbl_ddl_Pur_Configuration
3
Phase 3 — On First Add New (lazy load)
Fetch dropdown options for each dropdown column in the item grid via GetFilterDetail. Runs once; result is cached in component state.
4
Phase 4 — On Select Item Click
Pick item picker RB code by BasedOnID → fetch its RBID → fetch columns (GetDetailColData) → fetch item rows
5
Phase 5 — On Select Supplier Click
Fetch supplier list for selected Division via Fn_tbl_FetchCustomerSupplierTranWs4Web
6
Phase 6 — On Insert Items (Indent wise only)
POST selected indent rows to Fn_tbl_FetchIndentSummaryItem4Inquiry → receive aggregated parent item rows
7
Phase 7 — On Save Click
POST master JSON + detail JSON + indent detail JSON to /API/TranFormSave/Post_RB_PurInquiryMst_Save
